## Cookbook

> **Add fields/logic to a model:** `_inherit = 'crm.lead'` as in
> `my_crm/models/crm_lead.py`. Find the parent first:
> `grep -rn "class Lead" odoo_src/.../addons/crm/models/`.

> **Extend a form/list view:** view inheritance via `xpath` as in
> `my_crm/views/crm_lead_views.xml`. Find the view id:
> `grep -rn "crm_lead_view_form" odoo_src/.../addons/crm/views/`.

> **Override a printed report:** QWeb template inheritance (`inherit_id`) as in
> `my_crm/report/sale_order_templates.xml`. Find the template:
> `grep -rn "sale.report_saleorder_document" odoo_src/.../addons/sale/`.

> **Register a scheduled action:** `ir.cron` data record as in
> `my_crm/data/ir_cron_data.xml`. See existing crons:
> `grep -rn "ir.cron" odoo_src/.../addons/crm/data/`.

> **Override front-end behavior:** OWL component inheritance as in
> `my_crm/static/src/components/`. Find the parent component:
> `grep -rn "class ActivityMenu" odoo_src/.../addons/mail/static/src/`.

## Decision tree

1. **Configuration?** Can Odoo do this through Settings / Technical menus / data
   records? Verify via live introspection (see below) before writing code. Most
   requests end here.
2. **Custom code?** Find the change-type in the Cookbook above, use the map to
   locate the owning module, grep for the parent symbol, follow the example.
3. **Third-party module?** Flag as a lead to investigate separately — out of scope
   for this reference.

### Live introspection (requires a running instance or restored dump)

```bash
docker compose run --rm odoo shell -d <db>
```

```python
# Is a module installed?
env['ir.module.module'].search([('name', '=', 'crm'), ('state', '=', 'installed')])
# Does a config param exist?
env['ir.config_parameter'].get_param('crm.default_probability')
# Does a field already exist on a model?
env['ir.model.fields'].search([('model', '=', 'crm.lead'), ('name', '=', 'x_custom_score')])
